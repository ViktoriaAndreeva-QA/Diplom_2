- pytest 7.4.0

- pytest-cov 4.1.0

## Тестовые сценарии

### Создание пользователя (`/api/auth/register`)
- Создание уникального пользователя
- Ошибка при создании уже существующего пользователя
- Ошибка при отсутствии обязательных полей (email/password/name)

### Авторизация пользователя (`/api/auth/login`)
- Успешный вход под существующим пользователем
- Ошибка при входе с неверными данными (email/password)
- Ошибка при входе под несуществующим пользователем
- Ошибка при входе с пустыми полями

### Создание заказа (`/api/orders`)
- Создание заказа с авторизацией
- Создание заказа без авторизации (баг API)
- Создание заказа с разными наборами ингредиентов
- Создание заказа с двойными порциями
- Ошибка при создании заказа без ингредиентов
- Ошибка при создании заказа с невалидным хешем ингредиента


## Запуск всех тестов
py -m pytest tests -v
py -m pytest tests --cov=. --cov-report=term-missing -v


## Allure-отчёт
pytest tests --alluredir=allure-results -v
allure generate allure-results -o allure-report --clean
allure open allure-report


## Найденные баги
Создание заказа без авторизации
Ожидаемый статус: 401
Фактический статус: 200
Тест помечен как @pytest.mark.xfail


## Примечания
- Константы разделены по типам: urls, responses в разных файлах
- Тестовые данные вынесены отдельно: ингредиенты в папке data/
- Вспомогательные функции в helpers.py: генерация данных, форматирование
- Фикстуры только для сложной логики: создание/удаление пользователей
- Параметризация тестов: проверка множества сценариев
- Автоудаление тестовых данных: пользователи удаляются после тестов
